ds-free-api 项目完整分析报告
一、项目概览
ds-free-api 是一个用 Rust 编写的 API 代理服务器（v0.2.5，edition 2024），它将 DeepSeek 内部协议包装为标准的 OpenAI 和 Anthropic 兼容 API，实现"免费使用 DeepSeek 模型"的目标。

核心技术栈：

Rust 1.95.0（rust-toolchain.toml 锁定版本）
axum 0.8.8 — HTTP 服务器框架
reqwest 0.13.2 — HTTP 客户端（含 json/stream/multipart 特性）
wasmtime 44.0.0 — 执行 DeepSeek 的 PoW WASM 求解器
tiktoken-rs 0.11.0 — 客户端 prompt token 计数
tokio 1.51.0 — 异步运行时（含 signal 特性用于优雅关闭）
dashmap 6 — 无锁并发 HashMap（账号池）
pin-project-lite 0.2.17 — 流式响应包装

二、分层架构（自底向上）
2.1 ds_core/ — DeepSeek 核心层
这是最底层，直接与 DeepSeek API 交互。

模块	文件	职责
客户端	client.rs	原始 HTTP 调用，零业务逻辑。封装了登录、session CRUD、PoW 挑战、completion 流、文件上传/查询、WASM 下载。关键设计：Envelope<{code, msg, data: {biz_code, biz_msg, biz_data}}> 统一解析 DeepSeek 响应信封。
账号池	accounts.rs	1 账号 = 1 session = 1 并发。AccountGuard RAII 模式：获取时标记 Busy，Drop 时自动释放。选号策略：选空闲时间最长的账号（最大化冷却间隔）。支持动态增删账号、自动恢复（每 60s 重登 Error 账号）、连续 3 次失败标记 Invalid。
PoW 求解	pow.rs	基于 DeepSeek 的 WASM（DeepSeekHashV1）求解 PoW 挑战。关键创新：通过签名动态探测 wasm-bindgen 导出符号（__wbindgen_add_to_stack_pointer、allocator、wasm_solve），避免 WASM 更新后硬编码符号失效。
聊天编排	completions.rs	v0_chat() 是核心入口：获取账号 → 分割历史 → 创建 session → 上传文件 → 计算 PoW → 发送 completion → 返回 GuardedStream。GuardedStream 将账号守卫与 SSE 流绑定，流结束时自动释放账号。
2.2 openai_adapter/ — OpenAI 协议适配层
这是中间层，将 OpenAI 兼容请求转换为 ds_core 内部格式，并将响应转回 OpenAI 格式。
